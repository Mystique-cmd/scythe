Workflow Detector - Interactive Test Bench

Simulate orchestrated network interactions to verify real-time detection heuristics.

How to verify:
  1. Open Chrome Developer Tools (Press F12 or right-click and select Inspect).
  2. Switch to the Workflow Detector tab in the DevTools header.
  3. Click any of the scenarios below to trigger multi-flow request patterns.
  4. Observe how they are sessionized, timed, and flagged in the DevTools panel.
Parallel Burst Low

Triggers rapid fan-out behavior by firing multiple fetches in parallel immediately after a user click.

Concurrent Check-Then-Act High

Runs multiple guarded validation+mutation flows in parallel to simulate racey concurrency on mutation endpoints.

Check-Then-Act Medium

Simulates a validation guard. First makes a GET request to verify session permissions, and only after completion executes a mutating POST request.

Multi-Domain Sync Low

Fires API operations across multiple target domains, flagging cross-origin service orchestration.

GraphQL Checkout High

Simulates a structured checkout pipeline executing multiple GraphQL mutations in sequence (e.g. createCart, submitPayment).

Faulty Transaction High

Triggers a multi-stage request flow where one database operation succeeds, but a subsequent action fails with a 500 error.

Staged Polling Loop Medium

Triggers an export job, then continually polls a task status endpoint in 300ms intervals until the job returns complete.

🛡️ Logic Flaw Detection Scenarios

Click to simulate vulnerable request patterns. Open the DevTools panel and view the Logic Flaws tab to see findings.

🔑 IDOR — Sequential Object IDs High

Simulates sequential user ID references in API paths (e.g. /api/users/1001, /api/users/1002) which enables enumeration of other users' data.

💰 Parameter Tampering — Price Manipulation High

Simulates a checkout attempt with a negative price value and a quantity of zero, exercising price/quantity manipulation detection.

🛂 Privilege Escalation — Client Role Control High

Simulates a registration/update request where the client supplies an isAdmin=true or role=admin parameter, which should be server-controlled.

📦 Mass Assignment — Protected Fields High

Simulates a profile update that sends internal fields like isVerified, balance, and role as part of the request body.

🔄 Process Bypass — Payment Without Cart High

Simulates a direct payment operation without a preceding cart/checkout step, bypassing the standard purchase workflow.

📝 Injection — SQL & NoSQL Patterns High

Simulates a login request with SQL injection pattern in email field and NoSQL operator in password field.

🏎️ Race Condition — Wide TOCTOU Window Medium

Simulates a check (GET /api/check-balance) followed by a mutation (POST /api/withdraw) with a 600ms gap, creating a large race window.
