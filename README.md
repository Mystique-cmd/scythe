# Browser DevTools Multi-Flow Workflow Detector

A Chrome developer tool extension (Manifest V3) that monitors network activity (XHR, fetch, and GraphQL requests) and correlates them with browser-side user interactions. It automatically analyzes request patterns to flag and diagnose multi-step, orchestrated backend workflows.

## Features
- **Sessionization**: Group incoming network requests into workflow clusters anchored to specific user events (clicks, form submissions, enter keypresses, page navigation).
- **Network Orchestration Timeline**: A visual representation of requests showing their start offset and duration relative to the user action.
- **Diagnostic Heuristic Engine**: Detects and flags:
  - **Sequential Fan-out**: Cascades of requests fired in rapid succession.
  - **Check-then-act Patterns**: GET validation endpoints executed followed by mutating operations.
  - **Cross-Domain Orchestration**: Request flows spanning multiple domains or service hostnames.
  - **Mixed Status Codes**: Successful requests clustered with failures (unstable flows).
  - **Staged Polling Loops**: Repetitive queries checking status/readiness endpoints.
  - **Pagination/Thresholding**: Tracking index parameters like `offset`, `limit`, `attempt` or `page`.
  - **Multiple GraphQL Mutations**: Multiple write actions clustered under a single user action.
  - **Mixed Telemetry**: Analytics and logging endpoints fired concurrently with core API requests.
- **GraphQL Parser**: Auto-parses query bodies, types (queries vs. mutations), operation names, and variables (supporting batched GraphQL endpoints).
- **Interactive Inspector**: Inspect detailed request/response headers, raw payloads, variables, and body values.
- **Integrated Test Bench**: Run mock client workflows inside the extension sandbox to preview heuristics immediately.

## Installation & Setup

1. **Download / Clone** this project folder to your local machine.
2. Open **Google Chrome** (or any Chromium browser).
3. Navigate to `chrome://extensions/`.
4. Enable **Developer mode** (toggle in the top-right corner).
5. Click **Load unpacked** in the top-left corner.
6. Select the `scythe` project root directory.

---

## How to Test and Verify

1. After loading the extension, click the **Details** page of the extension in `chrome://extensions/` or inspect any webpage.
2. Open a new tab and load the **Interactive Test Bench** page:
   - Click the **Test Bench** button inside the extension's DevTools panel, or open:
     `chrome-extension://<EXTENSION_ID>/src/ui/test_bench.html` *(replace `<EXTENSION_ID>` with the actual ID generated by Chrome)*.
3. Open **Developer Tools** on the Test Bench tab (`F12` or `Ctrl+Shift+I` / `Cmd+Option+I`).
4. Find the **Workflow Detector** tab at the top of the DevTools header panel.
5. Click the buttons on the Test Bench page to trigger different workflow simulation scenarios:
   - **Parallel Burst (Fan-out)**
   - **Check-Then-Act** (GET followed by mutating POST)
   - **Multi-Domain Sync** (requests sent to different hosts)
   - **GraphQL Checkout** (multiple GraphQL mutations containing variables)
   - **Faulty Transaction** (mixed status codes)
   - **Staged Polling Loop** (task start followed by three status checks)
6. Look at the **Workflow Detector** panel to inspect the results:
   - Expand individual items in the sidebar to review timeline diagrams.
   - Read diagnostic flags and see evidence of why a heuristic was matched.
   - Click on request entries in the HTTP Operations table to inspect request headers, GraphQL payloads, and responses.

## Configuration Settings
Click the **Gear** icon in the panel header to edit:
- **Association Window (ms)**: The threshold for linking network requests to a preceding click or keypress (default: `2500ms`).
- **Heuristic Severity Threshold**: Filter out minor findings (e.g. only show Medium/High alerts).
- **Preserve Log**: Retain session logs across browser navigation events.
